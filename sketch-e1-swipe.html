The swipe approach solves the capacity problem entirely. Each panel gets full width on mobile. The "Display" panel can hold as many pill groups as needed without cramping the selectors. Desktop shows both at once so nothing is hidden. The peek edge is the same scroll affordance already used for the fretboard.
Summary: Approaches Compared
Single card (current approach)
Selector row + pill row in one card. Fits 3+2x2 comfortably. Adding more requires icons or abbreviations. All controls always visible.

Swipe panels
"Setup" card (selectors) + "Display" card (pills). Side by side on desktop, carousel on mobile. Unlimited capacity per panel. Trade-off: "how" controls hidden behind swipe on mobile.

Key question: Do users need to see both setup and display options at the same time on mobile? If they set up once and tweak display while playing, the swipe is natural. If they bounce between both constantly, one card is better.
